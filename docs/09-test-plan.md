# 09. テスト計画書

## 1. 単体テスト(Vitest)

| 対象 | 検証内容 |
|---|---|
| 終了日時の自動計算 | 施設の固定利用時間を正しく加算するか、日付またぎ計算が正しいか(例:ランドリー 22:00→翌3:00) |
| 予約可能期間判定 | 当日〜設定日数先の境界値(設定値変更時も含む) |
| 重複予約判定 | `newStart < existingEnd AND newEnd > existingStart`の境界値(完全一致、部分重複、隣接(重複なし)) |
| 日付またぎ判定 | 開始日と終了日が異なる予約の判定・表示用フラグ算出 |
| ステータス自動更新 | 終了時刻経過時の「利用完了」への遷移(キャンセル済みは対象外であること) |
| 岩盤浴清掃ステータス | 利用完了時の自動「清掃中」化、清掃完了操作でのリセット |
| 清掃警告ロジック | 次予約開始30分前判定、開始時刻超過判定 |
| 権限判定 | 一般スタッフ/管理者それぞれが呼べる/呼べない操作の判定 |
| アナリティクス集計 | 日別・週別・月別の集計境界値、タイムゾーン変換の正しさ |

## 2. 結合テスト

| シナリオ | 検証内容 |
|---|---|
| 新規予約から一覧反映 | 登録後、一覧取得APIに即座に反映されるか |
| 予約変更 | 終了日時の再計算、履歴記録 |
| 予約キャンセル | ステータス変更、一覧からの除外(トグルOFF時) |
| 別施設への移動 | 終了日時再計算、確認モーダル相当のフロー、重複時の拒否 |
| 利用停止と既存予約の競合 | 重複予約一覧の取得、一括キャンセルの実行、履歴記録 |
| 自動利用完了 | 同期処理呼び出しで正しく遷移するか(冪等性含む) |
| 通知作成 | 15分前通知・清掃警告・各種通知の重複生成防止 |
| 履歴作成 | 各操作種別ごとにaudit_logsが正しく記録されるか |
| 管理者認証 | ログイン成功/失敗、セッションによる権限制御 |

## 4. 同時操作テスト

2つのブラウザコンテキスト(擬似的な2端末)から、同一施設・同一時間枠へほぼ同時に予約リクエストを送信し、**片方のみ成功し、もう片方はDB制約(`23P01`)由来のエラーメッセージを受け取る**ことを確認する。Playwrightの並列リクエストまたはVitestでの直接DB呼び出しテストの両方で検証する。

## 5. 非機能・その他の確認観点

- PCブラウザ(Chrome/Edge/Safari最新版)での表示・操作確認
- iPad(横向き)での表示・操作確認、タップ操作の誤操作防止確認
- 予約登録・変更の応答時間(通常2秒以内)、一覧表示(通常3秒以内)の簡易計測
- アクセシビリティ:色だけに依存しないラベル・アイコン表示の確認
- セキュリティ:CSRF/XSS/SQLインジェクションに対する基本的な確認(自動テスト+手動確認)
