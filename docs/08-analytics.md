# 08. アナリティクス要件書

管理者専用画面。PC・タブレット横向きでの閲覧を優先し、スマートフォン向けレイアウトは考慮しない。

## 1. 指標とグラフの対応

| 指標 | 推奨グラフ | 理由 |
|---|---|---|
| 施設ごとの予約件数 | 横棒グラフ | 施設数が少なく(6〜数十)、件数比較は横棒が最も読みやすい |
| 日別の予約件数推移 | 折れ線グラフ | 時系列の増減トレンドを把握しやすい |
| 週別の予約件数推移 | 折れ線グラフ または 棒グラフ | 週単位は棒グラフでも比較しやすい(切替可能にする) |
| 月別の予約件数推移 | 棒グラフ | 月ごとの絶対量比較に適する |
| 時間帯別の利用数 | ヒートマップ(施設×時間帯) | 施設ごとの繁忙時間帯を一覧で比較できる |
| 予約者区分別の割合(宿泊中/チェックイン前/チェックアウト後) | ドーナツグラフ | 全体に対する割合の把握に適する |

## 2. フィルター

- 集計期間(開始日〜終了日、または「今日/今週/今月/カスタム」のプリセット)
- 施設(複数選択可)
- 施設カテゴリ(複数選択可)
- 予約者区分(複数選択可)
- 予約ステータス(複数選択可)
- キャンセル済み予約の集計対象への含有トグル(ON/OFFで再集計)

## 3. 集計方針

- タイムゾーンはAsia/Tokyoに統一(UTC保存データを変換してから日次/週次/月次に区切る)。
- 「日別」は00:00〜23:59(Asia/Tokyo)を1日として区切る。
- 「週別」は月曜始まりを基本とする。
- キャンセル済みを含める場合、ステータス別の内訳(予約中/利用中/利用完了/キャンセル)も併記できるようにする。

## 4. 画面レイアウト方針

- 上部にフィルターパネル(横並び、PC/タブレット横向きを想定した幅)
- 主要指標をカード形式のサマリー(合計予約件数、施設別トップ、期間比較)で最上段に表示
- グラフ群は2カラムグリッドで配置し、ヒートマップのみ全幅表示
- データ量が多い場合はローディング表示、0件の場合は「該当データがありません」を表示

## 5. 実装候補ライブラリ

Recharts(棒グラフ・折れ線グラフ・ドーナツグラフ)を基本採用し、時間帯別ヒートマップのみ独自実装(CSS Grid + 色スケール)またはRecharts非対応部分を専用コンポーネントで補う。
